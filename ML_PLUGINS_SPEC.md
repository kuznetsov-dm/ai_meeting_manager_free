# ТЗ на разработку ML-плагинов (Semantic & Neural)

**Цель:** Реализовать обработку транскрипций с использованием векторных представлений (Embeddings) и математической статистики для глубокого понимания структуры и смысла разговора.

**Требования:**
*   Использовать библиотеку `sentence-transformers` (локальные модели).
*   Использовать `scikit-learn` / `numpy` для математических операций.
*   Обеспечить graceful degradation (если библиотеки нет, падать мягко).

---

## Плагин №1: `text_processing.semantic_segmenter` ("Смысловой Архитектор")

**Задача:** Разбить встречу на логические главы (Topics) не по паузам, а по смене смысла разговора.

**Алгоритм:**
1.  **Вход:** Очищенные предложения (через `utils.merge_segments_smart`).
2.  **Векторизация:** Преобразование каждого предложения в вектор 384-d (MiniLM).
3.  **Матрица Подобия:** Расчет косинусного сходства между соседними группами предложений (Sliding Window).
4.  **Детекция Границ (Segmentation):**
    *   Поиск локальных минимумов в графике сходства (алгоритм C99 или TextTiling).
    *   Использование `sklearn.cluster.AgglomerativeClustering` для объединения похожих соседей.
5.  **Именование:** Для каждого кластера (Темы) найти предложение-центроид (наиболее близкое к среднему вектору кластера) или извлечь ключевые слова через TF-IDF внутри кластера.

**Выход:** `edited.md`, разбитый на заголовки `## [MM:SS] <Смысловой Заголовок>`.

## Плагин №2: `text_processing.neural_extractor` ("Нейро-Референт")

**Задача:** Сделать *Extractive Summary* — выбрать 10-15% самых важных предложений, которые передают суть встречи.

**Алгоритм:**
1.  **Векторизация:** Получение векторов предложений.
2.  **Граф Смысла (LexRank Continuous):**
    *   Построение графа, где узлы — предложения, а ребра — косинусная близость векторов.
    *   Расчет Centrality (PageRank): выявление предложений, на которые "ссылается" (по смыслу) больше всего других предложений.
3.  **Diversity Filter (MMR):**
    *   Выбор топ-N предложений.
    *   Фильтрация похожих (если два предложения говорят об одном и том же, берем только одно).
4.  **Action Detection (Hybrid):**
    *   Векторный поиск по близости к эталонным фразам ("надо сделать", "задача").
    *   Комбинация с Regex из `utils.py`.

**Выход:** `edited.md` в формате "Краткая выжимка" (Bullets) + Action Items.
