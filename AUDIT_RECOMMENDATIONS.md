# Аудит и рекомендации по улучшению `text_processing` артефактов

Дата: 31.01.2026
На основе анализа файлов: `output/*.edited.md`, `docs/edit_processing_improvements_2026-01-30.md`, `docs/TZ_text_processing.notes_structurer_2026-01-31.md` и исходного кода плагинов.

## 1. Результаты аудита

Подтверждаются проблемы, описанные в документации и ТЗ:
1.  **Качество "Темы":** Строка `ТЕМА:` часто содержит мусорные слова ("период_декабря_2025_года_киевский", "вот_так_сейчас"). Это связано с бедным списком стоп-слов в `utils.py` и простым алгоритмом сборки темы.
2.  **Структура и объем:** Текущие плагины (`variant_b`) генерируют слишком много блоков и буллетов. Структура "Block X" не информативна.
3.  **Игнорирование источника:** Плагины работают с сырым `transcript.txt`, игнорируя более качественную разбивку из `asr_segments.json`.
4.  **Отсутствие выделения действий:** В текущих артефактах нет блоков `Action Items`, `Decisions`, что критично для бизнес-пользователя.

### 2.2. Улучшение токенизации
Функция `tokenize` теряет дефисы. Рекомендуется:
```python
def tokenize(text: str) -> List[str]:
    # Разрешаем буквы, цифры и дефис внутри слов (но не в начале/конце)
    raw = re.findall(r"(?:\b[A-Za-zА-Яа-я0-9]+(?:-[A-Za-zА-Яа-я0-9]+)*\b)", text.lower())
    return [token for token in raw if token]
```

### 3.2. Настройка алгоритмов (Config Tuning)
Для соответствия требованиям ТЗ ("3-12 секций", "конспект") рекомендуются следующие параметры для `VariantB` / `NotesStructurer`:

*   `changepoint_penalty`: **25.0** (было 8.0) — чтобы выделять только крупные смены тем.
*   `min_block_sentences`: **15** (было 8) — чтобы избежать мелких дробных блоков.
*   `bullets_per_block`: **4** (было 5).
*   `topic_keywords_max`: **7** (было 5).
*   `dedup_similarity`: **0.85** (было 0.78) — строже удалять повторы.

### 3.3. Экстракция Action Items (New Feature)
Добавить метод `_extract_actions` на основе регулярных выражений:

```python
ACTION_PATTERNS = [
    r"(?:нужно|надо|необходимо|следует|обязаны)\s+(?:сделать|подготовить|написать|позвонить|отправить|проверить)",
    r"(?:давай|давайте)\s+(?:мы\s+)?(?:сделаем|посмотрим|решим|обсудим)",
    r"(?:прошу|попрошу)\s+(?:вас|тебя)?",
    r"(?:срок|дедлайн)\s+(?:до|к)\s+",
    r"(?:я\s+)?(?:беру|возьму)\s+(?:на\s+себя|в\s+работу)",
]

def _extract_actions(self, sentences: List[str]) -> List[str]:
    actions = []
    for sent in sentences:
        for pattern in ACTION_PATTERNS:
            if re.search(pattern, sent, re.IGNORECASE):
                actions.append(sent)
                break
    return actions[:15] # Лимит по ТЗ
```
